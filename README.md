# sign-IN-UP
A Sign In/Sign Up system using Object-Oriented Programming  manages user registration and login through classes and objects. It uses methods like **signUp()** and **signIn()** to securely handle user authentication while applying encapsulation to protect user data. This approach makes the code organized, reusable, secure, and easy to maintain.

A Sign In/Sign Up system is an application that allows users to create a new account and access it securely. In Object-Oriented Programming (OOP), the system is designed using classes and objects, making the code more organized, reusable, and easier to maintain. The **User** class stores user information such as name, email, username, and password, while methods are used to perform different operations.

During the **Sign Up** process, a new user enters their personal information, including a username and password. The system validates the entered data, checks whether the username or email already exists, and saves the information if it is valid. This ensures that each user has a unique account.

During the **Sign In** process, an existing user enters their username or email and password. The system compares the entered credentials with the stored data. If the information matches, the user is successfully logged in; otherwise, an error message is displayed asking the user to try again.

The OOP concept of **Encapsulation** keeps user data private and allows access only through specific methods. **Inheritance** enables different user types, such as Admin and Customer, to share common features while adding their own functionalities. **Polymorphism** allows the same method to behave differently for different user types, and **Abstraction** hides unnecessary implementation details from the user.

Overall, the OOP-based Sign In/Sign Up system provides a secure, modular, and scalable solution for user authentication. It reduces code duplication, improves readability, simplifies maintenance, and makes it easier to add new features in the future.
